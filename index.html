1.1

System and Network Architecture Concepts in Security Operations

Explicação

Para atuar como analista de segurança, você precisa entender a arquitetura dos sistemas e redes que está protegendo. Isso inclui como os logs são gerados e coletados, como sistemas operacionais organizam dados sensíveis, e como a infraestrutura moderna (cloud, contêineres, zero trust) molda a superfície de ataque.

O objetivo 1.1 cobre os fundamentos que sustentam toda operação de segurança: sem entender onde os dados vivem, como a identidade é gerenciada e como o tráfego flui, é impossível detectar anomalias ou responder a incidentes de forma eficaz.

Tópicos principais
Log Ingestion

Coleta centralizada de logs de múltiplas fontes. Inclui sincronização de tempo (NTP) e níveis de logging (debug, info, warning, error, critical). Sem timestamps corretos, correlação de eventos é impossível.

OS Concepts

Windows Registry, hardening de sistema, estrutura de arquivos e processos. Analistas precisam saber onde malware costuma se esconder (ex: HKCU\Software\Microsoft\Windows\CurrentVersion\Run).

Infrastructure Concepts

Serverless, virtualização e contêineres alteram como os workloads são isolados. Um contêiner comprometido pode escalar para o host se o runtime não estiver hardened.

Network Architecture

On-premises, cloud, hybrid, segmentação de rede, zero trust, SASE e SDN. O modelo zero trust assume que nenhum usuário ou dispositivo é confiável por padrão, mesmo dentro da rede corporativa.

Identity & Access Management

MFA, SSO, Federation, PAM, passwordless, CASB. SSO reduz friction mas cria single point of failure — se a conta SSO for comprometida, todos os serviços associados ficam expostos.

Encryption

PKI e SSL inspection. SSL inspection (TLS interception) permite que firewalls de próxima geração inspecionem tráfego criptografado, mas exige gerenciamento cuidadoso de certificados.

Sensitive Data Protection

DLP (Data Loss Prevention), PII (Personally Identifiable Information) e CHD (Cardholder Data). PCI DSS protege CHD; GDPR/LGPD protege PII. DLP monitora e bloqueia exfiltração.

Exemplo prático
Cenário

Um analista percebe que logs do servidor web chegam ao SIEM com timestamps 3 horas adiantados em relação aos demais sistemas. Isso dificulta a correlação de eventos durante uma investigação de incidente.


Causa: O servidor não está sincronizado com o servidor NTP da organização. O nível de logging está em WARNING, então eventos de nível INFO (como logins bem-sucedidos) não estão sendo registrados.


Ação corretiva: Configurar NTP centralizado em todos os ativos e ajustar o logging level para INFO no servidor web para capturar eventos de acesso completos.
